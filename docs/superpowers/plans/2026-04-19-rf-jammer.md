# RF Jammer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Enable the second palette button (RF Jammer) — a soft-kill area-slow defense that halves ISR speed, barely slows OWA / Payload, stacks by taking the strongest slow, and pairs with Interceptor for the first visible "layered defense" moment.

**Architecture:** New `applyJamEffects(state)` runs before `updateDrones` and sets each drone's `speedMultiplier`. Motion helpers multiply speed by that multiplier. `updateDefenses` gains an explicit `rfJammer` branch (no-op for per-defense state). Placement gets a dashed range ring for RF Jammer; the defense sprite gets a per-type accent so jammer and interceptor are visually distinct.

**Tech Stack:** Vanilla JS (ES modules), HTML5 Canvas 2D. Manual verification in browser.

**Scope:** Exactly `docs/superpowers/specs/2026-04-19-rf-jammer-design.md`. **Out of scope:** pulsing-ring animation, real tower sprite, jammed-drone tint, sounds, Laser, HPM.

**Conventions:** same as prior plans — plain objects + pure functions, central gameState, dt seconds / config ms, each task ends runnable.

- [ ] **Step 1: Add `applyJamEffects` to `src/game/defenses.js`**

At the bottom of `src/game/defenses.js`, append:

```js
export function applyJamEffects(state) {
  const cfg = CONFIG.defenses.rfJammer;
  for (const d of state.drones) {
    if (d.hp <= 0 || d.phase === 'done') { d.speedMultiplier = 1; continue; }

    let minMult = 1;
    for (const def of state.defenses) {
      if (def.type !== 'rfJammer') continue;
      const dx = d.x - def.x;
      const dy = d.y - def.y;
      if (Math.hypot(dx, dy) > cfg.range) continue;
      const eff = cfg.effectivenessVs[d.type] ?? 0;
      const mult = 1 - (1 - cfg.slowFactor) * eff;
      if (mult < minMult) minMult = mult;
    }
    d.speedMultiplier = minMult;
  }
}
```
